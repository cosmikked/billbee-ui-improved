# Modes

Use these when the user signals a specific design action.

## craft

Handle the task end to end. Do a short `shape` pass first, then implement the UI changes in code.

## shape

Plan the surface before editing:

- define the register
- map primary task and supporting tasks
- list the important states
- identify the two or three decisive visual moves

If the user asked for implementation, continue directly into code after shaping.

## critique

Review the current UI and report findings first. Focus on UX clarity, hierarchy, spacing, typography, color, interaction states, responsiveness, and anti-patterns. Use file references when possible.

## audit

Check technical UI quality: accessibility, keyboard behavior, semantic structure, responsive breakpoints, loading states, empty states, error states, and performance-sensitive rendering choices.

## polish

Run the last 10 percent pass before shipping. Tighten spacing, labels, hierarchy, state completeness, alignment, empty states, and rough motion edges.

## bolder

Increase point of view. Push hierarchy, scale, contrast, layout character, or color commitment where the current design feels safe.

## quieter

Reduce noise. Remove competing accents, over-animation, excessive saturation, redundant labels, and decorative clutter.

## distill

Strip to the essence. Remove low-value components, repeated wrappers, duplicate copy, and unnecessary visual structure.

## harden

Make the UI production-ready. Add missing edge states, failure paths, overflow handling, long-copy handling, empty states, loading states, and i18n-aware spacing.

## onboard

Improve first-run experience: empty states, guided steps, contextual teaching, progressive disclosure, and next-action clarity.

## animate

Add purposeful motion. Use it for state change, feedback, and orientation. Avoid decorative choreography on task-heavy product surfaces.

## colorize

Introduce or refine color strategy intentionally. Decide whether the surface should stay Restrained or move toward a more committed palette.

## typeset

Improve font choice, scale, line length, weight contrast, and text rhythm.

## layout

Fix spacing rhythm, grouping, scan paths, density, alignment, and container strategy.

## delight

Add small memorable touches without reducing clarity or trust.

## overdrive

Push past conventional solutions when the brief supports it. Increase ambition, but keep the implementation coherent and shippable.

## clarify

Improve labels, helper text, empty-state copy, button text, and error messages. Remove vague nouns and repetitive filler.

## adapt

Tune the UI for device classes, viewport constraints, and responsive interaction changes.

## optimize

Remove UI performance problems caused by rendering volume, layout thrash, oversized effects, or heavy interaction code.
